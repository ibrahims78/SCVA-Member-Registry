# SCVA Design System

> اللغة البصرية الموحَّدة لتطبيق إدارة أعضاء الجمعية السورية لجراحة وأمراض القلب.  
> أيّ قيمة بصرية (لون، مقاس خط، تباعد، ظلّ) **لا** تُكتب مباشرة في المكوّن — تُقرأ من هذا النظام.

---

## 1. الفلسفة

ثلاثة مبادئ تحكم كلّ قرار بصري:

1. **الهدوء الطبّي** — لا ألوان صاخبة، التباين العالي محصور في النصّ والأفعال.  
2. **العربية كمواطن أوّل** — كلّ token مُختبَر في RTL قبل LTR.  
3. **الاتّساق فوق الإبداع المتفرِّق** — توسيع النظام أهمّ من ابتكار حلول لقطة لكل شاشة.

---

## 2. الألوان (Colors)

كلّ الألوان تُخزَّن كقيم HSL خام (`H S% L%`) ثمّ تُغلَّف بـ `hsl(var(--token))` عند الاستخدام.  
هذا يسمح بضبط الشفافية ديناميكياً: `hsl(var(--primary) / 0.5)`.

### 2.1 الوضع الفاتح

| Token | القيمة | الاستخدام |
|---|---|---|
| `--background` | `210 25% 98%` | خلفية الصفحة الأساسية |
| `--foreground` | `222 47% 11%` | النصّ الأساسي |
| `--card` | `0 0% 100%` | سطح البطاقات والحاويات |
| `--card-foreground` | `222 47% 11%` | نصّ داخل البطاقات |
| `--primary` | `196 90% 32%` | لون العلامة التجاريّة، الأفعال الرئيسيّة |
| `--primary-foreground` | `210 40% 98%` | نصّ فوق `primary` |
| `--primary-hover` | `196 90% 28%` | حالة hover للأفعال الرئيسيّة |
| `--secondary` | `210 30% 95%` | أزرار/شارات ثانويّة |
| `--muted` | `210 25% 95%` | خلفيات هادئة (Inputs، Sections) |
| `--muted-foreground` | `215 16% 42%` | نصوص تفسيريّة |
| `--accent` | `196 80% 94%` | تمييز ناعم (badges، حالات active) |
| `--accent-foreground` | `196 90% 28%` | نصّ فوق `accent` |
| `--destructive` | `0 78% 52%` | حذف، أخطاء حرجة |
| `--success` | `152 56% 38%` | اشتراك مدفوع، حفظ ناجح |
| `--warning` | `38 92% 48%` | اشتراك مستحق، تنبيه |
| `--info` | `210 90% 52%` | معلومات إعلاميّة |
| `--border` | `214 26% 88%` | حدود ناعمة |
| `--input` | `214 26% 88%` | حدود حقول الإدخال |
| `--ring` | `196 90% 40%` | حلقة التركيز |
| `--sidebar` | `210 25% 99%` | خلفية الشريط الجانبي |
| `--sidebar-accent` | `196 80% 94%` | البند النشط في الشريط الجانبي |

### 2.2 الوضع الداكن
نفس الـ tokens مع قيم مُعتَّمة. انظر `:root.dark` في `client/src/index.css`.

### 2.3 قاعدة التباين
- نصّ عادي مقابل خلفيته: ≥ **4.5:1** (WCAG AA).  
- نصّ كبير (≥18px غامق): ≥ **3:1**.  
- لا يُستخدم أيّ زوج لون بدون التحقّق من هذه النسبة.

## 3. الخطوط (Typography)

- **العائلة:** `Cairo` للعربية أوّلاً، `Inter` كاحتياط للإنجليزية.  
- **النسبة:** 1.25 — مريحة للنصّ العربي الذي يحتاج ارتفاعات سطر أكبر.

| Token | px | الاستخدام |
|---|---|---|
| `--text-xs` | 12 | تسميات صغيرة، `Badge` |
| `--text-sm` | 14 | نصّ ثانوي، meta |
| `--text-base` | 16 | النصّ الأساسي |
| `--text-lg` | 18 | عناوين فرعيّة |
| `--text-xl` | 20 | عناوين بطاقات |
| `--text-2xl` | 24 | عناوين أقسام |
| `--text-3xl` | 30 | عناوين صفحات |
| `--text-4xl` | 36 | hero / login / فارغ كبير |

> **قاعدة:** لا يستخدم أيّ مكوّن أكثر من **3 مستويات حجم** في نفس الشاشة.

## 4. الزوايا (Radii)

| Token | px | الاستخدام |
|---|---|---|
| `--radius-sm` | 6 | حقول، badges |
| `--radius-md` | 8 | الافتراضي للأزرار والبطاقات |
| `--radius-lg` | 12 | البطاقات الكبيرة، المودالات |
| `--radius-xl` | 16 | hero blocks |
| `--radius-full` | ∞ | الصور الرمزيّة، النقاط |

---

## 5. الظلال (Elevation)

| Token | الاستخدام |
|---|---|
| `--shadow-xs` | فصل بصري ناعم جداً (rows, dividers) |
| `--shadow-sm` | البطاقات في حالتها الطبيعيّة |
| `--shadow-md` | البطاقات عند hover، Popovers |
| `--shadow-lg` | المودالات، القوائم المنسدلة الكبيرة |
| `--shadow-focus` | حلقة التركيز الموحّدة |

> **قاعدة:** لا تُستخدم أكثر من طبقة ارتفاع واحدة على نفس الشاشة في نفس اللحظة (مثلاً مودال `lg` يُغلق `md` على البطاقات تحته).

---

## 6. التباعد (Spacing)

نستخدم سلّم Tailwind القياسي (مضاعفات 4px). القواعد المُلزِمة:

| المساحة بين | الحدّ الأدنى |
|---|---|
| عنوان صفحة وأوّل عنصر تحته | `space-y-6` (24px) |
| البطاقات في شبكة | `gap-4` (16px) |
| داخل البطاقة (header → content) | `space-y-3` (12px) |
| العناصر الإدارية (toolbar) | `gap-2` (8px) |

---

## 7. الحركة (Motion)

- **مدّة قصيرة:** `150ms` (تبديل حالة زر، hover).  
- **مدّة متوسّطة:** `250ms` (دخول مودال، فتح قائمة).  
- **مدّة طويلة:** `400ms` (انتقال بين صفحات، stagger).  
- **منحنى:** `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out طبيعي).
- **احترام `prefers-reduced-motion`:** كلّ الحركات تُعطَّل تلقائياً.

أداة جاهزة في `index.css`: كلاس `.stagger-fade-in` يُطبَّق على حاوية فيُحرّك أوّل ٦ أبناء بتأخير متتالٍ.

---

## 8. التركيز وإمكانية الوصول

حلقة تركيز موحّدة عبر `:focus-visible` في `index.css`:

```css
outline: 2px solid hsl(var(--ring));
outline-offset: 2px;
```

- `:focus-visible` فقط (لا تظهر مع كلِك الفأرة، تظهر مع التنقّل بلوحة المفاتيح).
- العرض موحّد عبر التطبيق بصرف النظر عن نوع العنصر.

## 9. كيفيّة استخدام الـ Tokens

### في React + Tailwind v4
```tsx
// ✅ صحيح
<button className="bg-primary text-primary-foreground hover:bg-primary-hover">
  حفظ
</button>

// ❌ خطأ — قيمة مباشرة
<button style={{ background: '#096B8F' }}>حفظ</button>
```

### في الرسوم البيانيّة (Recharts)
```tsx
import { tokens } from "@/lib/design-tokens";

<Bar fill={tokens.chart[0]} />  // أوّل لون من سلسلة العلامة
```

### قاعدة الاستثناء الوحيدة
يُسمح بقيمة hex مباشرة فقط في:
- ملف `index.css` نفسه (تعريف الـ tokens).
- ملفات SVG ثابتة لا تتغيّر مع الوضع (مثل شعار الجمعيّة الرسمي).

---

## 10. الطباعة

كلّ شيء في `@media print` داخل `index.css`:
- خلفية بيضاء، نصّ أسود، حدود رماديّة فاتحة.
- إخفاء أيّ عنصر له `data-print="hide"` أو كلاس `.no-print`.
- إظهار العناصر التي لها كلاس `.print-only`.
