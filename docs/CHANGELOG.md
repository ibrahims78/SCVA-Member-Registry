# سجل التغييرات — SCVA Members System

## الإصدار 1.1.0 — تحديث نماذج الأعضاء واستيراد البيانات والنسخ الاحتياطي

**التاريخ:** 2026-04-28  
**النطاق:** واجهة المستخدم، قاعدة البيانات، واجهة برمجة التطبيقات (API)

---

### 1. تخفيف متطلبات نموذج إضافة العضو

#### المشكلة
كان نموذج إضافة عضو جديد يُلزم المستخدم بملء جميع الحقول (14 حقلاً) قبل الحفظ، مما يُصعّب إدخال بيانات غير مكتملة.

#### التغييرات المُنجزة

**`shared/schema.ts`**  
- تم تعديل مخطط قاعدة البيانات (Drizzle ORM) لإزالة قيد `notNull()` من الأعمدة الاختيارية:  
  `fullName`, `fatherName`, `englishName`, `birthDate`, `gender`, `specialty`, `email`, `phone`, `workAddress`, `city`, `joinDate`, `membershipType`
- تم تحديث مخطط Zod الخاص بـ `insertMemberSchema` لجعل جميع الحقول اختيارية باستثناء `firstName` و`lastName`.
- تم تنفيذ `npm run db:push` لمزامنة التغييرات مع قاعدة البيانات.

**`client/src/pages/AddMember.tsx`**  
- تم تحديث مخطط Zod المحلي ليعكس الحقول الإلزامية الجديدة (الاسم الأول والكنية فقط).
- تم إزالة علامة النجمة `*` من جميع الحقول الاختيارية في الواجهة.
- تم تعديل رسالة التوجيه في رأس الصفحة لتوضيح أن الحقول المتبقية اختيارية.
- تم تعديل منطق ملء النموذج عند التعديل ليتعامل مع القيم الفارغة (`null` / `undefined`) بأمان.

#### النتيجة
يمكن إضافة عضو جديد بمجرد إدخال **الاسم الأول والكنية**، ويمكن إكمال البيانات الأخرى لاحقاً.

### 2. استيراد بيانات الأعضاء من Excel

#### الوصف
إضافة ميزة استيراد جماعي للأعضاء عبر ملف Excel، مع توليد نموذج رسمي للتعبئة.

#### التغييرات المُنجزة

**`server/routes.ts`**  
- إضافة endpoint جديد: `POST /api/members/import`
  - يستقبل مصفوفة JSON تحتوي على بيانات الأعضاء.
  - يتحقق من كل صف بواسطة `insertMemberSchema` قبل الحفظ.
  - يعيد تقريراً تفصيلياً بعدد النجاحات والإخفاقات مع رسائل خطأ مفصّلة.
  - محمي بصلاحية المستخدم المُصادق عليه (`requireAuth`).

**`client/src/pages/Settings.tsx`**  
- إضافة قسم "استيراد بيانات الأعضاء" في صفحة الإعدادات يتضمن:
  - **تحميل النموذج**: يولّد ملف Excel جاهزاً بأعمدة مُسمّاة واضحة وصفوف توضيحية (مثال + تعليمات).
  - **رفع الملف**: يقرأ ملف Excel المُعبّأ ويحوّله إلى JSON ثم يرسله إلى الـ API.
  - **عرض النتائج**: يُظهر تقرير الاستيراد مباشرة في الواجهة (عدد النجاحات، الإخفاقات، ورسائل الخطأ).
- تم استخدام مكتبة `xlsx` (موجودة مسبقاً في المشروع) لتوليد وقراءة ملفات Excel.

#### أعمدة نموذج الاستيراد

| العمود | إلزامي | القيم المسموحة |
|--------|--------|----------------|
| الاسم الأول | نعم | نص حر |
| الكنية | نعم | نص حر |
| الاسم الكامل | لا | نص حر |
| اسم الأب | لا | نص حر |
| الاسم بالإنجليزية | لا | نص حر |
| تاريخ الميلاد | لا | `YYYY-MM-DD` |
| الجنس | لا | `male` أو `female` |
| التخصص | لا | `cardiology` أو `cardiac_surgery` |
| البريد الإلكتروني | لا | بريد صالح |
| رقم الهاتف | لا | نص حر |
| المدينة | لا | نص حر |
| عنوان العمل | لا | نص حر |
| تاريخ الانضمام | لا | `YYYY-MM-DD` |
| نوع العضوية | لا | `original` أو `associate` |
| معرّف الجمعية الأوروبية | لا | نص حر |

### 3. النسخ الاحتياطي الكامل للبيانات

#### الوصف
إضافة ميزة تصدير نسخة احتياطية شاملة تحتوي على جميع بيانات النظام.

#### التغييرات المُنجزة

**`server/routes.ts`**  
- إضافة endpoint: `GET /api/backup`
  - يتطلب صلاحية المدير (`requireAdmin`).
  - يجمع بيانات: الأعضاء، الاشتراكات السنوية، والمستخدمين (بدون كلمات المرور).
  - يُضيف بيانات وصفية: رقم الإصدار (`version`) وتاريخ التصدير (`exportedAt`).
  - يُرسل الملف مباشرةً كـ JSON قابل للتحميل باسم مُعرَّف بالتاريخ.

**`client/src/pages/Settings.tsx`**  
- إضافة قسم "النسخ الاحتياطي" في صفحة الإعدادات يتضمن:
  - شرح ما تتضمنه النسخة الاحتياطية.
  - زر لتصدير النسخة وتحميلها فوراً كملف JSON.
  - تغذية راجعة بالحالة (جارٍ التصدير / اكتمل).

#### بنية ملف النسخة الاحتياطية
```json
{
  "version": "1.0",
  "exportedAt": "2026-04-28T19:00:00.000Z",
  "data": {
    "members": [...],
    "subscriptions": [...],
    "users": [...]
  }
}
```
